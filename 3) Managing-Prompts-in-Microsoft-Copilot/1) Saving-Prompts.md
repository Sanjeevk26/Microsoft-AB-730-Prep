# Managing Prompts and Conversations in Microsoft 365 Copilot

## 1. Overview

Microsoft 365 Copilot provides capabilities to manage prompts and conversations effectively. Users can:

- Create and edit prompts
- Save prompts for reuse
- Share or schedule prompts
- Access prompt history and gallery

---

## 2. Creating a Prompt

A prompt can be entered in Copilot Chat.

### Example

"What is the AB 738 exam offered by Microsoft?"

- The prompt appears at the top
- The response is displayed below

---

## 3. Prompt Options

Each prompt has multiple actions available:

### 3.1 Edit

- Copies the existing prompt into the input box
- Does not modify the original prompt
- Allows creating a revised version

---

### 3.2 Copy

- Copies the prompt to clipboard
- Can be reused using paste (Ctrl + V)

---

### 3.3 Schedule

- Allows scheduling the prompt to run later

---

### 3.4 Save Prompt

- Saves the prompt to the Copilot Prompt Gallery

---

### 3.5 Copy Prompt Link

- Generates a shareable link for the prompt

---

## 4. Response Options

Options available for responses include:

- Read aloud
- Open in Microsoft Word
- Copy response
- Like or dislike response
- Share prompt and response
- Edit in Pages
- View sources used in the response

---

## 5. Saving Prompts

When saving a prompt:

- A title must be provided (max 35 characters)
- The prompt content is stored in the gallery
- The prompt itself cannot be edited during save

### Example

Title: "Microsoft AB 730"

---

## 6. Copilot Prompt Gallery

The Prompt Gallery can be accessed via:

- "View prompts"
- "See More" in a new chat
- Gallery icon (sparkle icon)

---

### 6.1 Suggested Prompts

- Provides pre-built prompt ideas
- Can be filtered by:
  - Task
  - Job type
  - Copilot agent

---

### 6.2 Your Prompts

- Displays user-saved prompts
- Shows:
  - Prompt title
  - Creator
- Allows:
  - Selecting prompts for reuse
  - Sharing prompts
  - Removing prompts

Note:
Removing prompts may require sign-out and sign-in to reflect changes.

---

## 7. Using Saved Prompts

- Selecting a saved prompt loads it into the editor
- Users can modify the prompt before execution
- Changes do not affect the original saved version

---

## 8. Prompts with Uploaded Files

### Scenario

- A file is uploaded during a conversation
- Prompt uses that file

### Limitation

When saved:
- The prompt does NOT include uploaded files
- Running the saved prompt without re-uploading the file may result in incorrect answers

---

## 9. Prompts with Referenced Work Content

### Scenario

- A prompt references a file from OneDrive or SharePoint

### Example

"Using the file about the AB 830 exam, what is the name of the exam?"

### Behavior

- The reference is retained in the saved prompt
- The file itself is not stored, only the reference
- Copilot can access the file if permissions exist

---

## 10. Key Difference: Uploaded Files vs References

| Feature                  | Uploaded File | Referenced Work Content |
|--------------------------|--------------|--------------------------|
| Saved with prompt        | No           | Yes (reference only)     |
| Requires re-upload       | Yes          | No                       |
| Uses SharePoint/OneDrive | No           | Yes                      |
| Persistent across chats  | No           | Yes                      |

---

## 11. Best Practices

- Use references instead of uploads for reusable prompts
- Keep prompt titles concise and meaningful
- Verify file availability when reusing prompts
- Use the prompt gallery for quick access
- Modify prompts at runtime instead of editing saved versions

---

## 12. Key Takeaways

- Prompts can be edited, copied, scheduled, saved, and shared
- Saved prompts are stored in the Copilot Prompt Gallery
- Uploaded files are not retained in saved prompts
- References to work content are retained and reusable
- Prompt gallery improves efficiency and consistency
