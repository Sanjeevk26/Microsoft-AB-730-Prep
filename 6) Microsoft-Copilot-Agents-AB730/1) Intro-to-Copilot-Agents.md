## Repository: Microsoft-Copilot-Agents-AB730

---

## 1. Overview

This section introduces **Copilot Agents**, which extend the standard chat experience into more structured, reusable, and capability-driven interactions.

---

## 2. Chat Experience vs Agent Experience

### 2.1 Chat Experience

In a typical Copilot chat:

- User provides a prompt  
- Copilot generates a response  

Example:

"What is the weather in New York?"

Follow-up:

"What is the weather tomorrow?"

Copilot uses:
- Conversation history (context)
- Work data (files, emails)
- Web data
- Custom instructions
- Memory

---

### 2.2 Agent Experience

Agents provide a more structured approach.

They include:

- Instructions (specific to the agent)
- Knowledge (data sources)
- Memory (contextual understanding)
- Capabilities (tools and actions)

Key difference:

- Chat → General-purpose, session-based  
- Agent → Purpose-built, reusable, configurable  

---

## 3. Creating an Agent (Copilot Studio Lite)

### Access

- Click **Create Agent**
- Go to **Configure tab**

---

### Components of an Agent

#### 3.1 Instructions

- Define how the agent should behave
- Applies only to that agent (not global)

---

#### 3.2 Knowledge

- Add data sources:
  - Files
  - SharePoint
  - Websites

---

#### 3.3 Memory

- Context stored for the agent
- Improves continuity of responses

---

#### 3.4 Capabilities

Examples:

- Code Analyzer:
  - Data analysis
  - Mathematical plotting
  - Generate Word, Excel, PowerPoint files

- Image Generator:
  - Create images from prompts

---

## 4. Copilot Studio Lite vs Copilot Studio Full

### Copilot Studio Lite

- Embedded in Copilot
- Simple agent creation
- Basic configuration

---

### Copilot Studio Full

- Separate platform
- Advanced capabilities:
  - Topics (structured workflows)
  - Conditions and variables
  - Multi-step conversations
  - Integration with Power Automate
  - Integration with external systems

---

### 6.1 Researcher

  - Unstructured data (Word, PowerPoint)

### 6.2 Analyst

- Works with:
  - Structured data (Excel, CSV)

  - Data analysis
  - Numerical insights

### Usage Limit

- Combined usage: up to 25 times per month (license dependent)

## 10. Agent Interaction Characteristics

| Feature            | Chat Experience | Agent Experience |
|--------------------|----------------|------------------|
| Structure          | Low            | High             |
| Reusability        | Limited        | High             |
| Customization      | Basic          | Advanced         |
| Workflow Support   | No             | Yes              |
| Tool Integration   | Limited        | Extensive        |

---

## 11. Managing Agents

### Actions Available

- Add agent
- Pin agent
- Share agent
- Uninstall agent

---

### Notes

- Uninstalling does not delete the agent
- Agent remains available in the store
- Can be reinstalled anytime

---

## 12. Best Practices

- Use built-in agents before creating new ones
- Choose agent based on data type:
  - Researcher → Unstructured data
  - Analyst → Structured data
- Use Prompt Coach to refine prompts
- Use Copilot Studio Full for complex workflows
- Validate outputs from agents

---

## 13. Key Takeaways

- Agents extend Copilot beyond simple chat
- Provide structured, reusable AI workflows
- Can integrate with data, tools, and automation
- Pre-built agents offer immediate value
- Advanced scenarios require Copilot Studio Full
